# S1 — Auth Service (Сервис аутентификации)

Сервис отвечает за регистрацию пользователей, вход по логину/паролю, выдачу токенов и сброс пароля. **Не хранит** профильные данные (ФИО, фото, контакты) — они управляются Profile Service.

> Поле `password` при сохранении хешируется и хранится как `pass_hash`. Все значения `username` и `email` автоматически приводятся к нижнему регистру перед сохранением.

---

## Зарегистрировать пользователя

Информация, требуемая для создания пользователя:

| Параметр  | Пояснение                                  | Обязательность | Тип    | Ограничение                      | Значение по умолчанию |
|-----------|--------------------------------------------|----------------|--------|----------------------------------|-----------------------|
| username  | Имя пользователя (приводится к нижнему регистру) | Да        | string | 3–50 символов, только a-z 0-9 _ | —                     |
| email     | Электронная почта (приводится к нижнему регистру) | Да       | string | формат email, до 100 символов    | —                     |
| password  | Пароль (хешируется перед сохранением)      | Да             | string | не менее 8 символов              | —                     |

**Уникальные комбинации параметров:**
- `username` — уникально глобально.
- `email` — уникально глобально.

> При занятом `username` или `email` возвращается ошибка 400.

Информация, возвращаемая при успешной регистрации:

| Параметр   | Тип               |
|------------|-------------------|
| id         | integer           |
| username   | string            |
| email      | string            |
| is_active  | boolean           |
| created_at | string (ISO 8601) |

---

## Войти (авторизация)

Информация, требуемая для входа:

| Параметр  | Пояснение         | Обязательность | Тип    | Ограничение         | Значение по умолчанию |
|-----------|-------------------|----------------|--------|---------------------|-----------------------|
| username  | Имя пользователя  | Да             | string | 3–50 символов       | —                     |
| password  | Пароль            | Да             | string | не менее 8 символов | —                     |

Информация, возвращаемая при успешном входе:

| Параметр   | Тип               |
|------------|-------------------|
| token      | string            |
| expires_at | string (ISO 8601) |

> Токен доступа действителен 24 часа.

---

## Обновить токен

Информация, требуемая для обновления токена:

| Параметр | Пояснение          | Обязательность | Тип    | Ограничение                | Значение по умолчанию |
|----------|--------------------|----------------|--------|----------------------------|-----------------------|
| token    | Действующий токен  | Да             | string | не пустой, существует в БД | —                     |

Информация, возвращаемая при успешном обновлении:

| Параметр   | Тип               |
|------------|-------------------|
| token      | string            |
| expires_at | string (ISO 8601) |

> Новый токен действителен 24 часа. Старый токен аннулируется немедленно после обновления.

---

## Деактивировать пользователя по ID

Информация, требуемая для деактивации:

| Параметр | Пояснение       | Обязательность | Тип     | Ограничение     | Значение по умолчанию |
|----------|-----------------|----------------|---------|------------------|-----------------------|
| id       | ID пользователя | Да             | integer | существует в БД | —                     |

Вернёт `true`, если пользователь был деактивирован (`is_active = false`), иначе `false`. Физически запись из БД не удаляется. Если пользователь уже деактивирован — вернёт `false`.

---

## Запросить сброс пароля

Информация, требуемая для запроса сброса пароля:

| Параметр | Пояснение          | Обязательность | Тип    | Ограничение   | Значение по умолчанию |
|----------|--------------------|----------------|--------|---------------|-----------------------|
| email    | Email пользователя | Да             | string | формат email  | —                     |

Информация, возвращаемая при запросе:

| Параметр | Тип     |
|----------|---------|
| success  | boolean |

> Метод всегда возвращает `success: true` независимо от наличия email в БД — это намеренное решение для безопасности (защита от перебора аккаунтов). Значение `true` не означает, что письмо будет отправлено. Токен сброса действителен 1 час.

---

## Сбросить пароль

Информация, требуемая для сброса пароля:

| Параметр | Пояснение         | Обязательность | Тип    | Ограничение               | Значение по умолчанию |
|----------|-------------------|----------------|--------|---------------------------|-----------------------|
| token    | Токен сброса      | Да             | string | действующий токен сброса  | —                     |
| new_pass | Новый пароль      | Да             | string | не менее 8 символов       | —                     |

Информация, возвращаемая при успешном сбросе:

| Параметр | Тип     |
|----------|---------|
| success  | boolean |

> При невалидном или истёкшем токене возвращается ошибка 400.

---

## Получить пользователя по ID

Информация, требуемая для получения пользователя:

| Параметр | Пояснение       | Обязательность | Тип     | Ограничение     | Значение по умолчанию |
|----------|-----------------|----------------|---------|------------------|-----------------------|

|------------|-------------------------|-------------------|

## Получить список пользователей по заданным параметрам

| Параметр  | Пояснение                                             | Тип     |
|-----------|-------------------------------------------------------|---------|
| is_active | Фильтр по активности                                  | boolean |
| search    | Поиск по username (частичное совпадение, без учёта регистра) | string  |
| limit     | Количество записей                                    | integer |
| offset    | Смещение                                              | integer |

Информация возвращается в виде списка пользователей, каждый содержит:

| Параметр   | Пояснение               | Тип               |
|------------|-------------------------|-------------------|
| id         | Идентификатор           | integer           |
| username   | Имя пользователя        | string            |
| email      | Электронная почта       | string            |
| is_active  | Активен ли пользователь | boolean           |
| created_at | Дата регистрации        | string (ISO 8601) |
